git init: Ініціалізує новий репозиторій Git в поточній папці.

git clone <url>: Створює локальну копію віддаленого репозиторію.

git add <файли>: Додає змінені або нові файли до "індексу", підготовлюючи їх до коміту.

git commit -m "Повідомлення коміту": Зберігає зміни, додані до індексу, як новий коміт разом з коротким повідомленням, що пояснює зміни.

git status: Виводить поточний статус робочого каталогу та індексу (які файли змінено, але ще не додано, та які файли готові до коміту).

git log: Виводить історію комітів репозиторію.

git pull: Стягує зміни з віддаленого репозиторію і об'єднує їх з вашим локальним репозиторієм.

git push: Відправляє збережені коміти до віддаленого репозиторію.

git branch: Виводить список гілок в репозиторії.

git checkout <гілка/коміт>: Переключає ваш робочий каталог на вказану гілку або коміт.

git merge <гілка>: Об'єднує зміни з вказаної гілки в поточну гілку.

git remote add <ім'я> <url>: Додає віддалений репозиторій з певним ім'ям та URL.

git remote -v: Показує список віддалених репозиторіїв, з якими ви працюєте.

git pull origin <гілка>: Стягує зміни з вказаної гілки від віддаленого репозиторію і об'єднує їх з вашою локальною гілкою.

git stash: Зберігає зміни в тимчасовому стеші, дозволяючи вам переключитися на іншу гілку без коміту змін.

git diff: Показує різницю між робочим каталогом і останнім комітом.

git diff <файл>: Показує різницю між вказаним файлом в робочому каталозі та останнім комітом.

git branch -d <гілка>: Видаляє локальну гілку. Використовуйте -D для видалення гілки, яка не була злита.

git push origin --delete <гілка>: Видаляє віддалену гілку на віддаленому репозиторії.

git checkout -b <нова_гілка>: Створює нову гілку та переключається на неї.

git reset <коміт>: Скасовує зміни, зроблені в після вказаного коміту, та повертає робочий каталог до стану вказаного коміту. Використовуйте з обережністю.

git reset --hard <коміт>: Аналогічно попередньому, але також скидає стан індексу.

git cherry-pick <коміт>: Додає зміни з вказаного коміту в поточну гілку.

git rebase <гілка>: Перебазовує поточну гілку на вказану гілку, включаючи всі коміти з цієї гілки.

git remote show origin: Показує інформацію про віддалений репозиторій "origin".

git log --oneline: Виводить історію комітів у короткому форматі.

git tag <ім'я_тегу>: Створює мітку (тег) на поточному коміті.

git config: Налаштування Git, такі як ім'я користувача, електронна пошта, тощо.

git fetch: Стягує зміни з віддаленого репозиторію, але не злиттяє їх з локальним репозиторієм.

git log -p: Виводить історію комітів з деталями змін (патчами).

git log --graph: Виводить історію комітів в графічному вигляді, показуючи гілки та їх злиття.

git blame <файл>: Показує, хто та коли вніс зміни до конкретного файлу.

git config --global alias.<скорочення> "<команда>": Створює скорочення для команди Git.

git clean -n: Виводить список файлів та папок, які будуть видалені за допомогою git clean -f.

git cherry: Показує коміти, які ще не були злиті з іншої гілки.

git grep "<пошуковий_вираз>": Шукає текст у файлах репозиторію.

git remote rename <старе_ім'я> <нове_ім'я>: Перейменовує віддалений репозиторій.

git stash list: Показує список стешів.

git config --global user.name "Ваше_ім'я": Встановлює ваше ім'я для комітів.

git config --global user.email "ваша_пошта@example.com": Встановлює вашу електронну пошту для комітів.

git commit --amend: Редагує останній коміт, додаючи нові зміни або редагуючи повідомлення.

git show <коміт>: Виводить деталі та зміни, внесені в конкретний коміт.

git remote prune origin: Видаляє локальні гілки, які відсутні на віддаленому репозиторії.

git reflog: Виводить журнал посилань, дозволяючи відновити втрачені коміти або гілки.

git bisect: Допомагає знаходити коміт, в якому виникла помилка, шляхом бінарного пошуку.

git submodule: Дозволяє робити підмодулі в репозиторії.

git log --author="Автор": Виводить історію комітів, зроблених вказаним автором.

git tag -a <ім'я_тегу> -m "Повідомлення": Створює анотований (з повідомленням) тег.

git revert <коміт>: Створює новий коміт, який скасовує зміни, внесені в вказаний коміт.

git mv <старий_файл> <новий_файл>: Перейменовує або переміщує файл та автоматично оновлює Git.
